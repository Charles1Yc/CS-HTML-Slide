Topic Intro

Code that has already been written

We have been using the System.out.println() method to print to the screen.

This method is part of the Java API (Application Programming Interface, 应用程序接口).

A method (方法) is a block of code that performs a specific task.

APIs and Libraries

Specification and reusable code

APIs are connected to libraries (库), which are collections of classes written by other programmers that provide a set of methods that can be used to perform specific tasks.

An API specification tells the programmer how to use those classes with methods in a library.

APIs and libraries are essential because they allow you to use code that has already been written by others.

Why APIs Matter

Focus on the task you are solving

Using code written by others saves time and allows you to focus on the specific task you are trying to accomplish.

The terms library, API, and package are often used interchangeably to mean similar things, but the textbook separates them this way:

  • a library is a collection of classes or code
  • an API describes how to use the library
  • a package is how Java organizes a library to be used

Packages

Groups of related classes

Classes in APIs and libraries are grouped into packages (包).

A package is a collection of related classes and interfaces that can be imported into a program to be used.

A package is like a folder of classes in a library’s file directory and is used to avoid name conflicts.

java.lang Package

Built into Java

Later on, we will learn to import other Java packages to use in our code.

There is a package that is already built into Java called java.lang that we are already using.

The java.lang package contains built-in classes and interfaces that are fundamental to the Java programming language, such as:

  • the String class
  • the System class used in System.out.println

Documentation

Attributes and behaviors of a class

Documentation found in API specifications and libraries is essential to understanding the attributes and behaviors of a class defined by the API.

The System class has an object called out that is type PrintStream.

The PrintStream class has a method called println() that we use to print to the screen.

Trace the Call

System.out.println

The textbook connection is:

System class
    has object out
        whose type is PrintStream
            which has println() methods

There is one println() method for each type that it can print.

Trace Check

How many println methods are there in the PrintStream class?

Answer: 10

Only count the println methods, not print or printf methods.
