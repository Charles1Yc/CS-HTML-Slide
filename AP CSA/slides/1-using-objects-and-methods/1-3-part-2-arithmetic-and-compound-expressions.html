Expressions and Operators

One expression evaluates to one value

Arithmetic Expressions (算术表达式) consist of numeric values, variables, and arithmetic operators that evaluate to a single numerical value.

For example, the expression 2 + 3 evaluates to 5.

Java uses the standard mathematical operators for addition (+), subtraction (-), and division (/).

Arithmetic Operators

Java operator symbols

The multiplication operator is written as *, as it is in most programming languages.

You may have noticed that + was also used to combine String and other values into new Strings. More on this later.

  • + addition
  • - subtraction
  • * multiplication
  • / division
  • % remainder

Numeric Result Types

int and double

Arithmetic expressions can be of type int or double.

An arithmetic expression consisting only of int values will evaluate to an int value.

An arithmetic expression that uses at least one double value will evaluate to a double value.

Truncating Division

Decimal part thrown away

When you are doing division with two integers, you will get an integer result and the decimal part of the result will be thrown away.

This is called truncating division (截断除法).

If you want a double result, you should make at least one of the values in the expression a double, like 2.0.

Logic Error Context

Math errors can cause runtime failures

Math errors sometimes lead to runtime errors in code.

The textbook example notes that when the Hubble Space Telescope was launched to space in 1990, a math coding error in a formula caused it to point in the wrong direction.

Thorough testing is the only way to make sure there are no logic errors that will cause runtime errors in your code.

Divide by Zero

Integer division by zero

Another runtime error that is possible in math expressions is dividing by zero, which is undefined in mathematics.

An attempt to divide an integer by the integer 0 will result in an ArithmeticException in Java.

Try it in one of the active codes to see what happens.

Compound Expressions

Multiple operators

Expressions can be combined into compound expressions (复合表达式) with multiple operators.

When compound expressions are evaluated, operator precedence (运算符优先级) rules are used, just like when we do math.

Multiplication *, division /, and remainder % are done before addition + and subtraction -.

Parentheses

Grouping first

Anything in parentheses is done first.

It does not hurt to put in extra parentheses if you are unsure as to what will be done first or just to make it more clear.

  • parentheses first
  • then *, /, %
  • then +, -
