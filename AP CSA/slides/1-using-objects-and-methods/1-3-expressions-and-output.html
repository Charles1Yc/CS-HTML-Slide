Topic Intro

Output and expressions

This topic connects two core skills:

  • using System.out.print and System.out.println to display information
  • evaluating arithmetic expressions (算术表达式) using Java operators

The main classroom habit is to predict output before running code.

Output

print versus println

Java has two common output methods:

  • System.out.println(value) prints the value followed by a new line.
  • System.out.print(value) prints the value without advancing to the next line.

"Hi there!" is a string literal (字符串字面量): zero or more characters enclosed in double quotes.

Code Task

Keep the output on one line

Run the starter code, then change it to print Hi there! on one line while keeping all three print statements.

public class MyClass
{
    public static void main(String[] args)
    {
        System.out.print("Hi ");
        System.out.println("there");
        System.out.print("!");
    }
}

Target output:

Hi there!

Literals

Fixed values in code

A literal is the code representation of a fixed value.

  • "Hi there!" is a string literal.
  • 100 is an integer literal.
  • 2.54 is a decimal numeric literal.
  • true and false are boolean literals.

Changing a literal in code changes the fixed value the program uses.

Escape Sequences

Special characters inside strings

Use a backslash when a character needs special treatment inside a string.

public class TestEscape
{
    public static void main(String[] args)
    {
        String message =
                "Here is a backslash quote \" "
                        + "and a backslashed backslash (\\) "
                        + "Backslash n \n prints out a new line.";
        System.out.println(message);
    }
}

AP CSA escape sequences here: \", \\, and \n.

Expressions and Operators

One expression evaluates to one value

An arithmetic expression combines numeric values, variables, and operators into one numerical result.

  • + addition
  • - subtraction
  • * multiplication
  • / division
  • % remainder

Java uses * for multiplication because keyboard character sets do not rely on a multiplication sign.

Truncating Division

int / int gives an int

An expression with only int values evaluates to an int. Integer division throws away the decimal part.

  • 2 / 3 evaluates to 0
  • 2.0 / 3 evaluates to 0.6666666666666666
  • if at least one value is double, the expression evaluates to double

Change the last line of the previous program to use 2.0 / 3.

Logic Error Task

Centimeters to inches

This code compiles and runs, but it uses the wrong operation.

public class CmToInches
{
    public static void main(String[] args)
    {
        System.out.print("100 centimeters in inches is: ");
        System.out.println(100 * 2.54);
    }
}

Fix the logic error. One inch equals 2.54 centimeters, so the expression should divide by 2.54.

Divide by Zero

A math error can become a run-time error

Dividing an integer by zero is undefined. In Java, attempting integer division by zero results in an ArithmeticException.

public class DivideByZero
{
    public static void main(String[] args)
    {
        System.out.println(3 / 0);
    }
}

This is a run-time failure, not a normal output value.

Compound Expressions

Operator precedence

Expressions can be combined into compound expressions (复合表达式). Java uses precedence rules:

  • parentheses first
  • then *, /, %
  • then +, -
  • operators with the same precedence are evaluated left to right

Extra parentheses are allowed when they make the intended grouping clearer.

Code Task

Predict before running

public class TestCompound
{
    public static void main(String[] args)
    {
        System.out.println(2 + 3 * 2);
        System.out.println((2 + 3) * 2);
        System.out.println(2 + (3 * 2));
    }
}

Expected output:

8
10
8

Explain why the first and third expressions match.

Remainder Operator

% gives the leftover part

The operator % returns the remainder after integer division.

Long division remainder

  • 5 % 2 evaluates to 1
  • 8 % 2 evaluates to 0
  • 3 % 4 evaluates to 3

With positive operands, the terms remainder and mod often give the same result for AP CSA work.

Remainder Pattern

Smaller left operand

If x is smaller than y, then x % y evaluates to x.

Reason:

  • y goes into x zero times.
  • The whole value of x is left over.
  • Therefore 2 % 3 evaluates to 2.

Use this shortcut carefully only when both values are positive in AP-style questions.
