Topic Intro

Why Java?

What do Android phones, Minecraft, and Netflix have in common? They all use Java in real software systems.

  • Java is a programming language (编程语言) used worldwide to create software.
  • AP CSA uses Java to study program structure, logic, and problem solving.
  • App Inventor apps can be translated to Java before they run on phones or tablets.

Algorithms

Step-by-step problem solving

Algorithms (算法) define step-by-step processes for completing a task or solving a problem.

  • A recipe is an algorithm for cooking a meal.
  • Directions are an algorithm for getting to a place.
  • In computer science, algorithms can sort, search, calculate, or control a game.

Sequencing (顺序执行) means steps are completed one at a time in a specific order.

Planning Code

English, diagrams, and pseudocode

Algorithms can be represented using:

  • written language
  • diagrams or flow charts
  • pseudocode (伪代码), simplified code written for planning

The classroom test is practical: each step should be precise enough to become a line of code.

First Java Program

Class and main method

Every Java program is written as a class (类). A method (方法) is a block of code that performs a specific task.

public class MyClass
{
    public static void main(String[] args)
    {
        // Put your code here!
    }
}

The Java run-time starts execution in the main method (主方法).

Compiling and Running

Source file, compiler, IDE

  • Java source code is saved in a source file such as MyClass.java.
  • If using your own files, the class name and file name must match.
  • A compiler (编译器) checks code for some errors and translates Java source files into class files.
  • An IDE (集成开发环境) provides tools to write, compile, and run code.

Errors detectable by the compiler must be fixed before the program can run.

Java Keywords

Lowercase keywords, capitalized class names

Keywords are reserved words with special meaning in Java.

  • public, class, static, and void must be lowercase.
  • System and String are class names and are capitalized.
  • A complete action line is a statement (语句).
  • Most Java statements end with a semicolon ;.

Syntax Errors

Debugging is normal

A syntax error (语法错误) means the program does not follow Java’s writing rules.

  • missing semicolon ;
  • open curly brace { without matching }
  • open quote " without matching close quote
  • wrong capitalization, such as system instead of System

The process of finding and removing errors is debugging (调试).

Reading Error Messages

Start with file, line, message, caret

FirstClass.java:5: error: unclosed string literal
       System.out.println("Hi there!);
                          ^
1 error
  • FirstClass.java gives the file.
  • 5 gives the line where the compiler detected the problem.
  • unclosed string literal says a string was opened but not closed.
  • ^ points near where the compiler noticed the problem.

The actual fix may be on the same line, earlier, or a little later.

Run-Time Errors

Compiles first, fails while running

A run-time error happens while the program is executing. An exception interrupts normal execution.

public class DivideByZero
{
    public static void main(String[] args)
    {
        System.out.println("It makes no sense to divide a number by zero!");
        System.out.println(3 / 0);
    }
}

After compilation succeeds, 3 / 0 causes an ArithmeticException.

Logic Errors

Code can run and still be wrong

A logic error is a mistake in the algorithm or program that causes incorrect behavior.

  • The program may compile successfully.
  • The program may run without crashing.
  • Testing with specific data is how you detect the wrong outcome.

This is why precise algorithms matter before writing Java code.

Comments

Notes for humans, skipped by the compiler

/* MyClass.java
   Programmer: My Name
   Date:
*/

int max = 10; // this keeps track of the max score
  • // starts a single-line comment.
  • /* and */ surround a multi-line comment.
  • Comments should explain intent or context, not repeat obvious syntax.
