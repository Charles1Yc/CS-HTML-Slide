Topic Intro

Methods and procedural abstraction

Up until now, all of our code has been in the main method, but complex programs are made up of many methods.

When faced with a large problem, we can break it down into smaller subproblems and solve each subproblem separately.

Each of these subproblems can be a method.

Method

Named block of code

A method (方法) is a named block of code that performs a task when it is called.

A block of code is any section of code enclosed in curly braces like { }.

These named blocks of code have many names in different programming languages: functions, procedures, subroutines, etc.

In Java they are called methods.

Procedural Abstraction

Use without knowing every detail

Procedural abstraction (过程抽象) allows a programmer to use a method and not worry about the details of how it exactly works.

For example, we do not need to know how a car exactly works in order to drive.

We know that if we hit the brakes, the car will stop.

Similarly, we can use a method someone else wrote without knowing exactly how it works.

Why Methods

Organize and avoid repetition

We divide up a program into multiple methods in order to organize the code, reduce its complexity, and avoid repetition of code.

The song “Old MacDonald Had a Farm” has a lot of repetition.

Instead of writing the same code over and over in order to print the song, we can write methods that contain the repeated code and run them when needed.

Repeated Code Answer

Identical repeated lines

The repeated lines are:

  • System.out.println("Old MacDonald had a farm.");
  • System.out.println("E-I-E-I-O");
  • System.out.println("E-I-E-I-O");
  • System.out.println("Old MacDonald had a farm");
  • System.out.println("E-I-E-I-O");

When you see repeated code, that is a signal for you to make a new method.

Method Call

Name plus parentheses

A method call is when the code calls out the method’s name in order to run the code in the method.

The method call always includes the method name and parentheses ().

Sometimes there is data inside the parentheses if the method needs that to do its job.

// Method call
methodName();

Method Call Example

println()

When we write:

System.out.println("Hello World");

we are calling the println() method to print out "Hello World".

Method calls are how Java runs code stored inside a named block.

Flow of Control

Method calls interrupt sequential execution

Execution in Java always begins in the main method in the current class.

Then, the flow of control (控制流) skips from method to method as they are called.

When a method ends, it returns to the line right after the method call.

Old MacDonald with Methods

codelens:: songvizOldMcdonald

public class OldMacDonaldSong
{
  public static void intro()
  {
    System.out.println("Old MacDonald had a farm");
    chorus();
  }

  public static void chorus()
  {
    System.out.println("E-I-E-I-O");
  }

  public static void main(String[] args)
  {
    intro();
    System.out.println("And on that farm they had a cow.");
    chorus();
    System.out.println("With a moo moo here and a moo moo there");
    System.out.println("Here a moo, there a moo, everywhere a moo moo");
    intro();
  }
}

Flow Trace

What happens in the song code

The main() method calls the intro() method, which calls the chorus() method.

When chorus() is called, flow skips to the chorus code, executes and prints "E-I-E-I-O", and then returns back to the method that called it.

The methods can be in any order, but programmers usually put the main method at the end after all the methods that it uses are defined.

public class Song
{

    {
        System.out.print("I like to ");
        eat();
        eat();
        eat();
        fruit();
    }

    public static void fruit()
    {
        System.out.println("apples and bananas!");
    }

    public static void eat()
    {
        System.out.print("eat ");
    }

    public static void main(String[] args)
    {
        print();
    }
}

Answer Reasoning

songMethodsTrace

Correct answer: D. I like to eat eat eat apples and bananas!

The print method calls eat three times and then calls fruit.

Figuring out what the code does is called tracing through the code.
