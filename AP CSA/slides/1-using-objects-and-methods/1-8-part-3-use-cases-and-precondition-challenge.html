Topic Intro

Preconditions support testing

Preconditions and postconditions help us test code and determine the validity (有效性) of software.

Software validity tests whether the software does what it is supposed to do before it is released.

This can be very important if the code is part of a satellite going to outer space or is going to be used in an emergency condition.

Good Testers

Try to break assumptions

Good software testers actually try to break the code.

They try all kinds of input to see what the software will do because you never know what users will try or what conditions there will be.

Always think what the preconditions of your code are, see what happens when you break them, and then see if you can protect or warn against that.

Use-Case Diagrams

Designing software systems

Preconditions and postconditions can also help us design better software systems.

Software designers often first draw a high-level Use-Case Diagram of a system that shows the different ways that a user might interact with a system before they build it.

A Use-case is a particular user interaction or situation in the system or software, and use-cases often become methods in the program.

Restaurant Use-Case Diagram

Customer, staff, and use-cases

Use-Case Diagram of a Restaurant System

The diagram shows 2 actors in the system: the customer and the staff at the restaurant, and 3 use-cases in circles.

Use-Case Conditions

Order Food

After drawing a Use-Case Diagram, designers write down the preconditions and the postconditions for each Use-Case.

Often the successful post-condition for one use-case becomes the preconditions for the next use-case.

  • Preconditions for “Order Food”: Customer enters restaurant. Staff is ready to take the order.
  • Postconditions for “Order Food”: Customer orders the food. Staff takes the order.

Use-Case Conditions

Eat Food

  • Preconditions for “Eat Food”: Customer has already ordered food. Staff has delivered food.
  • Postcondition for “Eat Food”: Customer eats the food.

Groupwork

Coding Challenge: Preconditions in Algorithms

Working in pairs or groups, come up with 4 steps that a user must do to purchase a product, for example a book on Java, in an online store.

List the preconditions and postconditions for each step.

You could pretend to buy something online to come up with the steps.

Do not forget to list the preconditions and postconditions for each step.
