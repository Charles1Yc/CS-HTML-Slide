Implementing Array Algorithms

Common algorithms with array traversal

In this lesson, you will study common algorithms using arrays and loops and practice FRQ (Free Response Question) problems.

Common algorithms that utilize array traversal include:

  • compute a sum or average of array elements
  • determine the minimum or maximum value in an array
  • search for a particular element in the array
  • test properties, pairs, duplicates, shifts, rotations, and reverse order

Accumulator Pattern

Sum and average

The accumulator pattern (累加器模式) is an algorithm that iterates through a set of values using a loop and updates an accumulator variable with those values.

For arrays, the accumulator pattern is used to compute the sum or average of the elements in the array.

The average is the sum divided by the number of elements in the array.

Accumulator Steps

Four-step pattern

  1. Initialize the accumulator variable before the loop.
  2. Loop through the values.
  3. Update the accumulator variable inside the loop.
  4. Print or use the accumulated value when the loop is done.
int[] values = {6, 2, 1, 7, 12, 5};
double sum = 0;
for (int val : values)
{
    sum += val;
}
double average = sum / values.length;
System.out.println("Average is " + average);

Min, Max, Search Algorithms

Loop with an if statement inside

Searching for the minimum or maximum follows a similar pattern of a loop with an if statement inside it.

The pattern is an enhanced for loop or an indexed for loop with an embedded if statement.

Remember that enhanced for loops cannot be used to modify primitive values in an array or to return an index.

for (int value : array)
{
    if (value < currentBest)
    {
        currentBest = value;
    }
}

Quick Check

mchoice:: mcq_array_error1

Given that array is an array of integers and target is an integer value, which best describes the conditions under which this code segment will return false?

public boolean find(int[] array, int target)
{
    for (int val : array)
    {
        if (val == target)
        {
            return true;
        }
        else
        {
            return false;
        }
    }
    return false;
}

Answer Reasoning

Return false too early

Correct answer: whenever the first element in array is not equal to target.

The loop returns as soon as checking the first element, whether it is equal to target or not.

It needs to check the whole array before returning false.

Quick Check

mchoice:: mcq_array_error2

Given that array is an array of integers and target is an integer value, which best describes the conditions under which this code segment will return true?

boolean temp = false;
for (int val : array)
{
    temp = (target == val);
}
return temp;

Answer Reasoning

Last value controls the result

Correct answer: whenever the last element in array is equal to target.

The variable temp is assigned on every loop iteration.

The last time through the loop it checks whether the last element is equal to target.
