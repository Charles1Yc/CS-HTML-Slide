ArrayList and its Methods

Resizable lists

A couple of lists

At the beginning of this unit, we learned about using arrays to hold collections of related data.

However arrays are not very flexible.

The size of an array is established at the time of creation and cannot be changed.

Why ArrayList

When size changes during runtime

What if you do not know how big the collection of data will be?

What if you want to both add and remove items from a collection?

For cases like this, Java has a class called ArrayList, a re-sizable list.

It stores items in an underlying array, tracks how many items have been added, and creates a bigger array when needed.

Mutable Size

ArrayList vs array

You can use ArrayList instead of arrays whenever:

  • you do not know the size of the array you need
  • you will add and remove items
  • you may need to change the array’s size dynamically during run time

An ArrayList is mutable (可变) in size and contains object references.

Quick Check

mchoice:: qloopList

Which of the following is a reason to use an ArrayList instead of an array?

Correct answer: An ArrayList resizes itself as necessary as items are added, but an array does not.

An ArrayList is really a dynamic array, one that can grow or shrink as needed.

import java.util.ArrayList

ArrayList is in java.util

The ArrayList class is in the java.util package.

A package (包) is a set or library of related classes.

Classes like String and Math are in java.lang; those classes are always available in any Java program.

Other packages, such as java.util, require importing or using the full class name.

Import Statements

Single class or wildcard

Import statements come before the class definition in a Java source file.

// Import just the ArrayList class from java.util
import java.util.ArrayList;

Another option is a wildcard import:

// Import everything in java.util including ArrayList
import java.util.*;

The source notes that AP CSA exam code will provide needed imports.

Quick Check

Correct answer: Import statements must be before other code in a Java source file.

You can have many import statements, and you do not need to import classes in java.lang.

Declaring and Creating ArrayLists

Generic type parameter

To declare an ArrayList, use:

ArrayList<Type> name

Type is called a type parameter and is the type of objects you want to store.

Programmers use the letter E and call it the generic type for an element.

String ArrayList

Declare and construct

// ArrayList<Type> name = new ArrayList<Type>();
// An ArrayList of Strings:
ArrayList<String> shoppingList = new ArrayList<String>();

Declaring an ArrayList variable does not create an ArrayList object.

It only creates a variable that can refer to an ArrayList or null.

To create the object, invoke a constructor such as new ArrayList<String>().

size()

Empty list size

The ArrayList() constructor constructs an empty list.

You can get the number of items in an ArrayList using the size() method.

A newly constructed ArrayList is empty and has size 0.

You cannot call methods on null; calling size on list2 in the exercise causes a NullPointerException.

ArrayLists of Numbers and Objects

Wrapper classes are required

You can create ArrayLists of integer and double values.

However, use wrapper classes Integer or Double as the type parameter because ArrayLists can only hold objects, not primitive values.

All primitive types must be wrapped in objects before they are added to an ArrayList.

This normally happens automatically thanks to autoboxing.

More Type Examples

Any kind of objects

You can put any kind of objects in an ArrayList, including instances of classes that you write.

// An ArrayList of Integers:
ArrayList<Integer> numList = new ArrayList<Integer>();
// An ArrayList of Student objects:
ArrayList<Student> roster = new ArrayList<Student>();

Quick Check

mchoice:: qArrayListInteger

Which is the correct way to create an ArrayList of integers?

Correct answer:

ArrayLists cannot hold primitive types like int; use the wrapper class Integer.
