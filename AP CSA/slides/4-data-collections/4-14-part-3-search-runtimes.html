Runtimes

Choosing between algorithms

How do we choose between two algorithms that solve the same problem?

They usually have different characteristics and runtimes, which measure how fast they run.

For the searching problem, it depends on your data.

Worst Case Behavior

Linear vs binary search

Binary search is much faster than linear search, especially on large data sets, but it can only be used on sorted data.

With searching, the worst case is usually if you cannot find the item.

With linear search, you would have to go through the whole array before realizing that it is not there.

Binary search is much faster even in this case because it eliminates half the data set in each step.

Runtime Table

Worst case comparisons

N Linear Search Binary Search
2 2 comparisons 2 comparisons
4 4 3
8 8 4
16 16 5
100 100 7

Every time we double the size of N, we need at most one more guess or comparison with binary search.

Big-O Note

Informal runtime comparison

For an array of size n, linear search runtime is a linear function.

Binary search runtime is a function of log base 2 of n.

This is called the big-O runtime function in computer science, for example O(log n) vs. O(n).

You do not need to know the log runtime growth function for the AP exam, but you should be able to calculate binary search steps by counting how many times you can divide by 2.

Quick Check

mchoice:: qbs_1

Which will cause the shortest execution of a binary search looking for a value in an array of integers?

Correct answer: The value is in the middle of the array.

If the value is in the middle, binary search returns after one iteration of the loop.

Quick Check

mchoice:: qbs_2

Which conditions must be true to search for a value using binary search?

I. The values in the array must be integers.
II. The values in the array must be in sorted order.
III. The array must not contain duplicate values.

Correct answer: II only.

The only requirement is that the values must be ordered.

Quick Check

mchoice:: qbs_3

How many times would the loop in binary search run for:

int[] arr = {2, 10, 23, 31, 55, 86};
binarySearch(arr, 55);

Correct answer: 2

It compares with index 2, then index 4, and then returns 4.

Quick Check

mchoice:: qbs_4

If you had an ordered array of size 500, what is the maximum number of iterations required to find an element with binary search?

Correct answer: approximately 9 times

You can divide 500 in half 9 times, or observe that 2^9 = 512, which is slightly bigger than 500.

Classroom Check

A complete answer should include

  • explain why linear search worst case checks every element
  • explain why binary search eliminates half the search space each step
  • use the runtime table to compare worst case steps
  • state that binary search requires sorted data
  • estimate binary search steps by repeated halving or powers of two
