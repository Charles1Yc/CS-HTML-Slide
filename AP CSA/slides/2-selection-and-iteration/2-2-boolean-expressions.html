Testing Equality

== and != return Boolean values

The relational operators == and != (not equal) can be used to compare values.

They return true or false boolean values (布尔值).

Note from the textbook:

  • one = sign changes the value of a variable
  • two == equal signs test if a variable holds a certain value without changing it

Object References

== can compare references to objects

We can also use == or != to test if two reference values, like Turtle and String objects, refer to the same object.

Turtle Reference Equality

If two reference variables refer to the same object like the turtle on the right in the image, the test with == will return true.

Relational Operators

Numeric comparisons

The relational operators below are used to compare numeric values or arithmetic expressions.

operator meaning
< Less Than
> Greater Than
<= Less than or equal to
>= Greater than or equal to
== Equals
!= Does not equal

Java uses compareTo and equals for comparing String values.

Relational Operator Notes

Remembering <, >, <=, and >=

Textbook notes:

  • Think of < and > as arrows where the pointy end should point to the smaller value.
  • A < expression is true if the smaller number is on the left.
  • A > expression is true only if the smaller number is on the right hand side.
  • Write <= and >= in the same order you say them in English: less than or equal to, greater than or equal to.

Boolean Expressions

Positive and negative tests

Boolean variables or expressions have true or false values.

Relational operators can compare two variables or compare a variable against a constant value or expression.

// Test if a number is positive
(number > 0)
// Test if a number is negative
(number < 0)

Testing with Remainder

% supports divisibility checks

The remainder operator % is useful in coding. These Boolean expressions can test whether a number is even, odd, or divisible by another number:

// Test if a number is even by seeing if the remainder is 0 when divided by 2
(number % 2 == 0)
// Test if a number is odd by seeing if there is a remainder when divided by 2
(number % 2 != 0)
// Test if a number is a multiple of x
(number % x == 0)

Remainder Warning

Odd numbers can be negative

Textbook warning: Java’s % is a remainder operator and not a true mathematical modulo operator.

Do not check if a number is odd with:

num % 2 == 1

Remainder Warning

Correct odd test

That expression works for positive odd values, but a negative odd number has remainder -1. Use:

num % 2 != 0
