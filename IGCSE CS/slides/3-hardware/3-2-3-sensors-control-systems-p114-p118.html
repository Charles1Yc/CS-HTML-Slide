Control Systems

Automatic action

Sensors are used in both monitoring and control applications.

In a control system, if the new data is outside the acceptable range, the computer/microprocessor sends signals to control devices, such as valves, motors, heaters or pumps.

Figure 3.50

Control path

Figure 3.50 Monitoring and control systems using sensors

Figure 3.50 is a simplification of the process. Use the right-hand side for control: sensor input, ADC if necessary, comparison with stored values, output to devices, and feedback from the changed process.

Control Sequence 1/2

Input and analysis

  • sensors send signals to the microprocessor/computer
  • the signals are converted to digital, if necessary, using an analogue to digital converter (ADC)
  • the computer/microprocessor analyses the data received
  • the data is checked against stored values

Control Sequence 2/2

Output and feedback

  • if the new data is outside the acceptable range, the microprocessor/computer sends signals to control valves, motors and other devices
  • a DAC may be needed later in the output route if the controlled device needs an analogue signal
  • an actuator may be used to operate a physical device
  • the output from the system affects the next set of inputs from the sensors

Feedback Loop

The changed process becomes the next input

The output from the control system changes the process being controlled.

That changed process then affects the next set of sensor readings.

This repeated route is the feedback loop.

Examples of Control Path

Figure 3.50 right-hand route

The control path follows the right-hand side of Figure 3.50:

  • sensors send signals to the microprocessor/computer
  • signals are converted to digital if necessary using an ADC
  • the microprocessor/computer analyses the data by checking it against stored values
  • if the new data is outside the acceptable range, the microprocessor/computer sends signals to control valves, motors or other devices
  • the output from the system affects the next set of inputs from the sensors

This last step is the feedback loop.

Monitoring and Control

The difference

Monitoring sends a warning message or activates an alarm; the microprocessor/computer has no effect on what is being monitored.

Control sends signals to devices such as valves, motors, heaters or pumps, so the process is changed automatically.

Examples of Control

Textbook examples

  • turning street lights on at night and off during daylight
  • controlling the temperature in a central heating or air conditioning system
  • chemical process control, for example maintaining temperature and pH
  • operating anti-lock brakes on a car when necessary
  • controlling the environment in a green house / greenhouse

Application Pattern

Same structure in each context

Every control application in the next deck follows the same answer pattern:

sensor → ADC → computer comparison → DAC if needed → actuator/output device → feedback loop.

The details change, but the control logic stays the same: name the sensor, the stored value or pre-set value, the comparison, the device controlled, and how the changed output affects later readings.

Classroom Check

Do not stop at the output

A strong answer should include:

  • the sensor and measured property
  • ADC before the computer if the sensor data is analogue
  • comparison with stored values or pre-set values
  • DAC and actuator where the output device needs them
  • the feedback loop, where the output affects the next sensor input
