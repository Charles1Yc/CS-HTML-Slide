Character Sets

ASCII code and Unicode

The ASCII code system (American Standard Code for Information Interchange) was set up in 1963 for use in communication systems and computer systems. A newer version of the code was published in 1986.

The standard ASCII code character set consists of \(7\)-bit codes (0 to 127 in denary or 00 to 7F in hexadecimal) that represent the letters, numbers and characters found on a standard keyboard, together with \(32\) control codes that use codes 0 to 31 in denary.

Uppercase and Lowercase Codes

Comparing characters in binary

Consider the uppercase and lowercase codes in binary of characters:

'a' 1100001   hex 61   (lower case)
'A' 1000001   hex 41   (upper case)

'y' 1111001   hex 79   (lower case)
'Y' 1011001   hex 59   (upper case)
  • the sixth bit changes from \(1\) to \(0\) when comparing the lowercase and uppercase of a character
  • this makes the conversion between the two an easy operation
  • the character sets (a to z, 0 to 9, and so on) are grouped together in sequence, which speeds up usability

Extended ASCII

8-bit character codes

Extended ASCII uses \(8\)-bit codes (0 to 255 in denary or 0 to FF in hexadecimal).

  • this gives another \(128\) codes to allow for characters in non-English alphabets
  • it also allows some graphical characters to be included

Unicode

Supporting all languages

ASCII code has a number of disadvantages. The main disadvantage is that it does not represent characters in non-Western languages, for example Chinese characters. As can be seen in Figure 1.6, DOS and Windows use different characters for some ASCII codes.

For this reason, different methods of coding have been developed over the years. One coding system is called Unicode (统一码):

  • Unicode can represent all languages of the world
  • there is overlap with standard ASCII code, since the first \(128\) characters are the same
  • Unicode can support several thousand different characters in total
  • ASCII uses one byte to represent a character, whereas Unicode will support up to four bytes per character

Unicode Consortium Goals

Version 1.0

The Unicode consortium was set up in 1991. Version 1.0 was published with five goals:

  • create a universal standard that covered all languages and all writing systems
  • produce a more efficient coding system than ASCII
  • adopt uniform encoding where each character is encoded as 16-bit or 32-bit code
  • create unambiguous encoding where each 16-bit and 32-bit value always represents the same character
  • reserve part of the code for private use to enable a user to assign codes for their own characters and symbols

Classroom Check

Understand character encoding standards

A complete answer should include:

  • that standard ASCII uses \(7\)-bit codes for the letters, numbers and characters found on a standard keyboard
  • that ASCII also includes \(32\) control codes
  • that Extended ASCII uses \(8\)-bit codes, giving another \(128\) codes
  • that ASCII has disadvantages because it does not represent non-Western languages
  • that Unicode can represent all languages of the world and can support several thousand different characters
  • that the first \(128\) Unicode characters are the same as standard ASCII
