Blockchain

Decentralised database

Blockchain (区块链) is a decentralised database.

All the transactions of networked members are stored on this database. The blockchain consists of a number of interconnected computers but they are not connected to a central server.

Transaction Copies

Stored on all computers

All transaction data is stored on all computers in the blockchain network.

Whenever a new transaction takes place, all the networked computers get a copy of the transaction; therefore it cannot be changed without the consent of all the network members.

This effectively removes the risk of security issues such as hacking.

Blockchain Uses

Areas listed in the textbook

Blockchain is used in many areas, such as:

  • cryptocurrency exchanges
  • smart contracts
  • research, particularly within pharmaceutical companies
  • politics
  • education

Block Structure

What a block contains

Whenever a new transaction takes place, a new block is created.

Part Meaning
Data name of sender and recipient, amount of money, and so on
Hash value a unique value generated by an algorithm, usually SHA 256, acting as a fingerprint
Previous hash value points back to a previous block in the chain

Hash Values

Unique to each block

A new hash value is created each time a new block is created.

This hash value is unique to each block and includes a timestamp, which identifies when an event actually takes place.

Block 1 is known as the genesis block since it does not point to any previous block.

Tamper Evidence

A changed block breaks the chain

If block 2 is changed in some way, any changes to the data within block 2 will cause the value of the hash to change.

This means block 3 and beyond will now be invalid since the chain was broken between block 2 and block 3.

This helps prevent tampering.

Proof-of-work

Slowing down block creation

Proof-of-work makes sure it takes ten minutes to determine the necessary proof-of-work for each block before it can be added to the chain.

This is policed by miners, special network users that get a commission for each new block created.

The whole process of creating new blocks is slowed down, which foils hackers and regulates the currency by all the network computers.

Adding New Blocks

Checked before joining the blockchain

When a new block is created, it is sent to each computer in the blockchain and is checked for correctness before being added.

If a new network user is created, they get a copy of everything in the whole blockchain system.

Activity 5.2

Textbook practice

The textbook asks students to:

  • draw a seven-block blockchain network
  • describe what would happen if block 4 was hacked to change the sum of money
  • explain the main differences between digital currency and cryptocurrency

Classroom Check

Blockchaining

A complete answer should include:

  • blockchain is a decentralised database storing all networked members’ transactions
  • all computers in the blockchain network store transaction data
  • each block contains data, a hash value and a previous hash value
  • changing a block changes its hash and makes later blocks invalid
  • proof-of-work slows block creation and is policed by miners
  • new blocks are checked by each computer before being added
