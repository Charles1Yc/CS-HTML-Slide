Symmetric Encryption

Same key for both operations

Symmetric encryption (对称加密) uses an encryption key; the same key is used to encrypt and decrypt the encoded message.

First, consider a simple system that uses a 10-digit denary encryption key (this gives 1 × 10¹⁰ possible codes). Suppose our encryption key is:

4 2 9 1 3 6 2 8 5 6

This means every letter in a word is shifted across the alphabet by +4, +2, +9, +1, +3, +6, +2, +8, +5, +6 places, repeating for the whole message.

Symmetric Encryption Example

COMPUTER SCIENCE IS EXCITING

Plaintext (top) and ciphertext (bottom) after applying the key 4 2 9 1 3 6 2 8 5 6:

C O M P U T E R   S C I E N C E   I S   E X C I T I N G
G Q V Q X Z G Z X   I M G W D   H O U M C   I M V R O J

To get back to the original message, apply the same decryption key but in reverse: each letter shifted −4, −2, −9, −1, −3, and so on.

For example: ‘G’ −4 → ‘C’, ‘Q’ −2 → ‘O’, ‘V’ −9 → ‘M’

Symmetric Encryption: Key Length

Modern encryption keys

However, modern computers could ‘crack’ this encryption key in a matter of seconds. To combat this, we now use 256-bit binary encryption keys that give 2²⁵⁶ (approximately 1.2 × 10⁷⁷) possible combinations.

(Even this may not be enough as we head towards quantum computers.)

Symmetric Encryption Problem

Key distribution challenge

The real difficulty is keeping the encryption key a secret (for example, it needs to be sent in an email or a text message which can be intercepted).

  • The security issue is always the main drawback of symmetric encryption
  • A single encryption key is required for both sender and recipient
  • This problem led to the development of asymmetric encryption

Asymmetric Encryption

Two keys: public and private

Asymmetric encryption (非对称加密) was developed to overcome the security problems associated with symmetric encryption. It makes use of two keys:

  • Public key (公钥) — made available to everybody
  • Private key (私钥) — only known to the computer user

Both types of key are needed to encrypt and decrypt messages.

The matching pairs of keys are mathematically linked but can’t be derived from each other.

Tom & Jane: Step 1 and 2

Asymmetric encryption in action

Suppose Tom and Jane work for the same company and Tom wishes to send a confidential document to Jane:

Step 1: Jane uses an algorithm to generate a matching pair of keys (private and public) that they must keep stored on their computers.

Step 2: Jane now sends her public key to Tom.

Tom                         Jane
                            Public key + Private key
Jane sends Tom              (matching pair)
her public key  ←-----------

Tom & Jane: Step 3 and 4

Encryption and decryption

Step 3: Tom now uses Jane’s public key to encrypt the document he wishes to send to her. He then sends his encrypted document (ciphertext) back to Jane.

Step 4: Jane uses her matching private key to unlock Tom’s document and decrypt it.

  • This works because the public key used to encrypt and the private key used to decrypt are a matching pair generated on Jane’s computer
  • Jane can’t use the public key to decrypt the message — only the matching private key works

Public Key Sharing

Multiple senders

Jane can also exchange her public key with any number of people working in the company, so she is able to receive encrypted messages from everyone:

Tom ----→ Public key ---
                         |
Susan ---→ Public key ---→ Jane
                         |   Public key
Mike ----→ Public key ---→ Private key
                         |
Megan ---→ Public key ---

All senders encrypt using Jane’s public key; only Jane can decrypt using her matching private key.

Two-Way Communication

Everyone needs their own key pair

If a two-way communication is required between all five workers: - They all need to generate their own matching public and private keys - All users then need to swap public keys so they can send encrypted documents between each other - Each worker will then use their own private key to decrypt information being sent to them

Symmetric vs Asymmetric Comparison

Comparing the two encryption methods

Aspect Symmetric Asymmetric
Keys One (shared) Two (public/private)
Speed Fast Slower
Key distribution Main drawback — keeping key secret Solved — public key shared openly
How it works Same key encrypts and decrypts Public key encrypts, matching private key decrypts
Main use Bulk data encryption Secure key exchange, where key distribution is the main issue

The key distribution problem is the central difference — asymmetric encryption was developed specifically to solve this.

Key Points Summary

Symmetric encryption

  • Uses one key for both encryption and decryption
  • Key must be kept secret — this is the main drawback
  • Modern systems use 256-bit binary keys (2²⁵⁶ possible combinations)
  • Example: 10-digit key 4291362856 shifts each letter by successive digits

Asymmetric encryption

  • Uses two mathematically linked keys — public (shared) and private (secret)
  • Public key encrypts → matching private key decrypts
  • Anyone with the public key can send encrypted messages to the key owner
  • For two-way communication, all parties need their own key pair and must swap public keys

Classroom Check

Understand encryption types and applications

A complete answer should include:

  • symmetric encryption: same key encrypts and decrypts — fast, but key distribution is the main security drawback
  • 256-bit binary keys give 2²⁵⁶ ≈ 1.2 × 10⁷⁷ possible combinations
  • asymmetric encryption: uses public key (shared) and private key (secret) — mathematically linked but can’t be derived from each other
  • process: Jane generates key pair → sends public key to Tom → Tom encrypts with public key → Jane decrypts with private key
  • public key can be shared with any number of people; all can send encrypted messages that only the private key holder can decrypt
  • for two-way communication, all parties need their own key pair and must swap public keys
