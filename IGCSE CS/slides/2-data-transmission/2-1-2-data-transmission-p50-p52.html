Data Transmission Factors

Three considerations when transmitting data

Data transmission can be over a short distance (e.g. computer to printer) or over longer distances (e.g. global network). Three factors need to be considered:

  • The direction of data transmission (can data transmit in one direction only, or in both directions?)
  • The method of transmission (how many bits can be sent at the same time?)
  • How data will be synchronised (how to make sure the received data is in the correct order?)

These factors are usually considered by a communication protocol.

Serial vs Parallel Transmission

Bit-by-bit vs multi-bit transmission

Data can be transmitted in two ways:

Serial data transmission (串行): - Data is sent ONE BIT AT A TIME over a SINGLE WIRE/CHANNEL - Bits are sent one after the other as a single stream - Works well over long distances - Data transmitted at a slower rate than parallel - Data will arrive at its destination fully synchronised (in the correct order) - Example: connecting a computer to a printer via a USB connection

Parallel data transmission (并行): - SEVERAL BITS OF DATA (usually 1 byte) are sent down SEVERAL CHANNELS/WIRES all at the same time - Each channel/wire transmits one bit - Faster method of data transmission than serial - Works well over short distances only - Example: internal circuits in a computer (very short distances between components, high-speed essential)

Note: Both serial and parallel data transmission can be simplex, half-duplex or full-duplex.

Data Skewing

Problem with parallel transmission over long distances

Over longer distances (for example, over 20 metres), parallel data can become skewed (数据偏移):

  • Data can arrive unsynchronised — bits arrive out of order
  • The longer the wire, the worse this can become
  • Serial transmission does not have this problem: data arrives fully synchronised because only one channel/wire is used

Simplex, Half-duplex, and Full-duplex

Communication directions

Mode Direction Example
Simplex One-way only Computer to printer, broadcast TV/radio
Half-duplex Both ways, but not simultaneously Walkie-talkies
Full-duplex Both ways simultaneously Broadband internet connection, telephone call

Simplex Transmission

One-way communication

Simplex (单工) occurs when data can be sent in ONE DIRECTION ONLY (for example, from sender to receiver):

  • Data flows in one direction only
  • An example would be sending data from a computer to a printer
  • Receiver cannot send data back

Half-duplex Transmission

Alternating two-way communication

Half-duplex (半双工) occurs when data is sent in BOTH DIRECTIONS but NOT AT THE SAME TIME:

  • Data can be sent from ‘A’ to ‘B’ and from ‘B’ to ‘A’ along the same transmission line
  • But they can’t both be done at the same time
  • An example would be a walkie-talkie where a message can be sent in one direction only at a time; but messages can be both received and sent

Full-duplex Transmission

Simultaneous two-way communication

Full-duplex (全双工) occurs when data can be sent in BOTH DIRECTIONS AT THE SAME TIME (simultaneously):

  • Data can be sent from ‘A’ to ‘B’ and from ‘B’ to ‘A’ along the same transmission line simultaneously
  • An example would be a broadband internet connection

Classroom Check

Understand transmission modes

A complete answer should include:

  • three factors: direction, method, synchronisation (via communication protocol)
  • serial: one bit at a time, single wire, works over long distances, no skewing
  • parallel: several bits, several wires, faster but limited to short distances, risk of skewed data (bits arrive out of synchronisation over >20m)
  • simplex: ONE DIRECTION ONLY (e.g. computer to printer)
  • half-duplex: BOTH DIRECTIONS but NOT AT THE SAME TIME (e.g. walkie-talkies)
  • full-duplex: BOTH DIRECTIONS AT THE SAME TIME (e.g. broadband internet)
  • both serial and parallel can be simplex, half-duplex or full-duplex
