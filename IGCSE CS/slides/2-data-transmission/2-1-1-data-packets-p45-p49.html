Data Packets

Breaking data into chunks

Data sent over long distances is usually broken up into data packets (数据包), sometimes called datagrams:

  • Large data broken into small chunks (typically 64 KiB)
  • Much easier to control than a long continuous stream of data
  • Each packet can be sent along a different route to its destination
  • Great benefit if a particular transmission route is out of action or very busy
  • The only obvious drawback is the need to reassemble the data when it reaches its destination

Packet Structure

Header, payload, and trailer

A typical packet is split up into three parts:

Packet header: - IP address of the sending device - IP address of the receiving device - Sequence number of the packet (allows correct reassembly) - Packet size (to check if all packets have arrived intact) - Note: header often also contains the total number of packets in this transmission

Payload: - The actual data being sent in the packet (usually about 64 KiB)

Packet trailer: - A way of identifying the end of the packet — essential to separate packets as they travel - An error checking method: Cyclic Redundancy Checks (CRC) are used

CRC Error Checking

Cyclic Redundancy Check in packet trailer

CRC (Cyclic Redundancy Check) (循环冗余校验) checks data packets:

  1. The sending computer adds up all the 1-bits in the payload
  2. This total is stored as a hex value in the trailer before sending
  3. Once the packet arrives, the receiving computer recalculates the number of 1-bits in the payload
  4. The computer then checks this value against the one sent in the trailer
  5. If the two values match, no transmission errors have occurred
  6. If they do not match, the packet needs to be re-sent

Packet Switching

Routing through networks

Packet switching (分组交换) is a method of data transmission in which a message is broken up into a number of packets. Each packet can then be sent independently from start point to end point.

At each stage in the transmission, there are nodes that contain a router (路由器). Each router: - Receives a data packet and, based on the information in the header, decides where to send it next - Determines which route the packet needs to take, using the destination IP address - Routing selection depends on the number of packets waiting to be processed at each node - The shortest possible path available is always selected — this may not always be the shortest possible path, since certain parts of the route may be too busy or not suitable

  • Packets can reach the destination in a different order to that in which they were sent
  • At the destination, packets will need to be reassembled into their correct order (using the sequence number sent in the header)

Benefits of Packet Switching

Advantages of this approach

  • No need to tie up a single communication line
  • Overcome failed, busy or faulty lines by simply re-routing packets
  • Relatively easy to expand package usage
  • High data transmission rate is possible

Drawbacks of Packet Switching

Limitations to consider

  • Packets can be lost and need to be re-sent
  • The method is more prone to errors with real-time streaming (e.g. a live sporting event transmitted over the internet)
  • There is a delay at the destination whilst the packets are being re-ordered

Router Role in Packet Switching

Moving packets between networks

Routers (路由器) make packet switching possible:

  • Examine destination IP in each packet’s header
  • Consult routing decisions to find next node
  • Forward packet to the next appropriate router
  • Make independent decisions for each packet
  • Thousands of routers interconnected across the internet
  • Packets hop from router to router (node to node) until reaching destination

Hopping: Preventing Lost Packets

How packets that never arrive are handled

Sometimes packets get lost because they keep ‘bouncing’ around from router to router and never actually reach their destination. This would eventually clog the network.

To overcome this, a method called hopping (跳数) is used:

  • A hop number is added to the header of each packet
  • This number is reduced by 1 every time the packet leaves a router
  • Each packet has a maximum hop number to start with
  • Once a hop number reaches zero, the packet is deleted when it reaches the next router
  • The missing packets will then be flagged by the receiving computer
  • A request to re-send these packets will be made

Note: hopping is not included on the syllabus but helps understand how packets can get lost.

Packet Reassembly

Reconstructing data at destination

Destination must rebuild original data:

  • Packets can reach the destination in a different order to that in which they were sent
  • Sequence numbers in the header identify the correct packet order
  • Missing packets detected (gaps in sequence numbers)
  • Reassembled into original order using sequence information
  • The receiving computer checks packet sizes to ensure all packets are intact
  • Transparent to user — happens automatically

Classroom Check

Understand packets and packet switching

A complete answer should include:

  • packets are small chunks (typically 64 KiB) — easier to control than continuous streams
  • packet header contains sender/receiver IP address, sequence number, packet size
  • packet trailer contains end-of-packet identifier and CRC error check
  • CRC: sending computer adds up all 1-bits in payload, receiver recalculates and compares
  • packet switching routes each packet independently; routers decide next hop by destination IP
  • benefits: no single line tied up, overcome failed lines, easy to expand, high data rate
  • drawbacks: packets can be lost, errors with real-time streaming, delay during re-ordering
  • hopping: hop number prevents lost packets clogging the network
