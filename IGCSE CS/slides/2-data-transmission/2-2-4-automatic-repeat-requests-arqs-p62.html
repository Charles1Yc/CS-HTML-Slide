Automatic Repeat Request (ARQ)

A third way to check data

ARQ (Automatic Repeat Request) (自动重复请求) is a third way used to check data following data transmission (after parity checks and echo checks).

ARQ uses positive and negative acknowledgements and timeout:

  • The receiving device receives an error detection code as part of the data transmission (this is typically a Cyclic Redundancy Check — CRC)
  • This is used to detect whether the received data contains any transmission errors
  • If no error is detected, a positive acknowledgement is sent back to the sending device
  • If an error is detected, the receiving device sends a negative acknowledgement and requests re-transmission of the data

ARQ: Timeout and Retransmission

Handling the case when no acknowledgement arrives

  • A timeout (超时) is used by the sending device — it waits a pre-determined amount of time
  • If no acknowledgement of any type has been received within this time limit, the sender automatically re-sends the data
  • This continues until a positive acknowledgement is received
  • Or until a pre-determined number of re-transmissions has taken place
  • ARQ is often used by mobile phone networks to guarantee data integrity

ARQ Process Summary

Complete ARQ flow

  1. Sender transmits data with error detection code (CRC)
  2. Receiver checks for errors using the error detection code
  3. If no error: receiver sends positive acknowledgement back to sender
  4. If error detected: receiver sends negative acknowledgement and requests re-transmission
  5. If no acknowledgement received within timeout period: sender automatically re-sends the data
  6. Process repeats until positive acknowledgement or pre-determined re-transmission limit reached

Classroom Check

Understand ARQ and retransmission

A complete answer should include:

  • ARQ uses positive/negative acknowledgements and timeout
  • receiver uses error detection code (CRC) to check for transmission errors
  • no error → positive acknowledgement sent to sender
  • error detected → negative acknowledgement and request for re-transmission
  • timeout: if no acknowledgement received within time limit, sender automatically re-sends
  • continues until positive acknowledgement or pre-determined re-transmission limit
  • ARQ is used by mobile phone networks to guarantee data integrity
